安卓手機內存（RAM）增加這麼快真的好麼？

首先你要懂得電子設備的運行原理。

電子設備無論桌面設備還是移動設備都是由 SoC(CPU、GPU 等)、RAM、儲存主要設備來組成的。

CPU 受限於架構、製作工藝、核心數量、核心頻率、L1/L2/L3 緩存大小、通用寄存器大小等影響，其中核心數量和核心頻率、製作工藝相比較其他要好提升（拋出硬件瓶頸理論）。

GPU 受限於製作工藝、核心數量、核心頻率。

RAM 受限於核心頻率和帶寬速度、帶寬接口大小、儲存容量等影響，其中儲存容量最好提升。

拋出桌面設備，移動設備 RAM 大小相比較“成本巨高”的 CPU 和 GPU 都要好提升，所以移動設備的 RAM 大小提升是最爲迅速的。

而操作系統的原理在於一個任務的計算是通過操作系統從儲存設備裏面去提取，這步體現出儲存讀寫速度的重要性，這也是桌面設備開機、運行重大任務固體硬盤要比機械鍵盤快很多的原因之一。然後任務會被放進 RAM，也就是所謂的內存，這步體現出 RAM 的帶寬速度，RAM 的大小可以理解爲一個木桶可以放多少的水，而帶寬速度一樣相當於水流的速度是固定的，大的 RAM 可以放進更多的任務，小的 RAM 放的任務就少，在 RAM 夠用的情況下，任務執行速度和系統運行的流暢度是相同的。這一點 Android 廠商在做宣傳的時候有點混淆消費者的傾向。由於 RAM 的特殊性，計算機在所有任務執行期間都是放進 RAM 去執行的，所以，大的 RAM 優勢可以放進更多的任務，可以加載更多的網頁內容等。下一步就是操作系統把任務放進 L3（很多 Android 設備的 CPU 並沒有）再逐步放進 L2 和 L1，期間所有的操作都是用的機器可以執行的低級代碼指令，這也是指令集的重要原因之一。L3 緩存大小重要性在整個 CPU 緩存系統裏面重要性最小，容量卻是最大，因爲在 RAM 裏面需要被 CPU 執行的任務會最先放進 L3 裏面，這也是爲什麼高通驍龍沒有 L3 的原因，因爲在移動設備裏面影響並不大，還可以減少成本和製作難度，然後 L2 和 L1 就比較重要了，其中最爲重要的是 L1，如果 L1 不大的話，你有再大的 RAM，任務一樣執行緩慢，就好比你人在排隊買票，售票口只有一個，分成再多的排也是無用的，所以 L1 緩存大小要比 RAM 重要的多！然後就是放進處理器去執行了，處理器的架構決定工作的效率，工藝決定着處理器的能耗和密度大小，核心數量和核心頻率決定執行速度，其中核心數量的作用在移動 CPU 裏面作用是最小的，研發難度卻是最低的。處理完的代碼會按照之前的順序逆向返回到 RAM 裏面，最後任務關閉或者需要存檔的數據會被放進儲存裏面，所有操作完成。

系統的流暢度主要體現在界面，界面流不流暢 GPU 佔很大的成分，不過現在的移動設備 GPU 都處於夠用的狀態，渲染出的幀數都足夠達到 60fps，完全滿足需求，只有在大型遊戲和視頻纔會體現好的 GPU 和差的 GPU 之間的差別。

RAM 在移動操作設備扮演的角色在於移動設備的 RAM 和顯存是合二爲一的，GPU 的計算也是在 RAM 裏面的，所以大一點 RAM 還是有好處的，RAM 過小肯定會影響到 GPU 的計算導致動畫掉幀。

現在移動設備發展（Android），4GB RAM 夠用，6GB RAM 成爲標配，8GB RAM 成爲新的旗艦配置，但更多的還是噱頭，因爲手機的 CPU 和 GPU 並沒有跟上 RAM 大小迅速擴大的腳步，一個車的發動機功率只有 200P 馬力，你給他配再大的油箱還是跑不過 300P 馬力的汽車，但是有可能跑的更遠，手機不同，在夠用的情況下，RAM 大小並不會提升流暢度，相反廠商不會標準的內存帶寬速度纔是系統流暢度的主要因素，但是帶來的影響就是能耗要更高。

最後發表下我的看法，Android 設備 RAM 發展的過快之前是因爲前期 Android 本身缺陷，現在 RAM 可以滿足 Android 的流暢運行，廠商是爲了在競爭當中更好的吸引用戶的目光，贏得競爭。這種行爲俗稱噱頭。
