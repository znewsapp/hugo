3.高分屏軟件適配

在知乎上，經常可以看到有人抱怨所謂“Windows 的高分屏適配問題”，聲稱在 Windows 上高分屏根本沒有必要，會產生界面模糊、縮小等問題。

這裏給大家澄清一下，高分屏可以帶來更加細膩的視覺體驗，很多人覺得 MacBook Pro 屏幕好，不過是因爲它有着 2560*1600 級別的分辨率，與他們見過的 1366*768 的筆記本相比高下立判。我建議大家在預算足夠的情況下，儘量購買分辨率在 1920*1080 以上的筆記本，至於軟件適配問題，請往下看。

1）系統層面的適配

Windows10 已經支持 200% DPI 縮放，也就是 Retina 所使用的縮放方式。設置界面在“設置——系統——顯示——更改文本、應用和其它項目的大小”。一般而言，系統會自動選擇合適的 DPI 縮放方案。

在線 PPI 計算器LUKIYA'S NEVERLAND

我的建議是 PPI 在 150 左右使用 125%縮放，在 200 左右使用 150%縮放，在 250 以上使用 200%縮放。正確地設置縮放比例可以讓系統界面正常顯示。

2）軟件層面的適配

所有 UWP 應用均支持高分屏幕

所有 UWP 應用均支持高分屏幕

所有 UWP 應用均支持高分屏幕

大部分國外 Win32 應用程序在 2015 年前後開始完成高分屏適配，請儘量安裝您所需軟件的最新版本。據本人所知，Adobe 全家桶在 CC 2014 版本後實現高分屏適配，Matlab 在 2015a 開始支持高分屏。請點擊查看原圖

而部分國產 Win32 應用程序則沒有做好適配，據本人所知騰訊在這一方面上算比較積極的，QQ 很早就完成了高分屏適配，但是國人常用的比如百度雲管家、迅雷等卻沒有做適配。

您可以選擇安裝同類 UWP 版本來解決這個問題。

如果您使用的系統縮放比例爲 125%，也可以在該應用圖標處右鍵點擊——屬性——兼容性——高 DPI 設置時禁用顯示縮放。這樣可以避免字體模糊的問題。

125%放大禁用縮放，影響不大。而 200%縮放直接不禁用縮放，由於是 4 對 1，模糊也不明顯。

4.關閉 Bitlocker 設備加密

BitLocker 使用 TPM 幫助保護 Windows 操作系統用戶數據，並幫助確保計算機即使在無人蔘與、丟失或被盜的情況下也不會被篡改。然而對於“普通用戶”而言，這項功能就像“查找我的 iPhone”一樣雞肋，被“莫名其妙”鎖住是更大的危險。所以如果您不是用這臺 PC 作爲工作機的話，關掉它吧。

五、XTU 控制 CPU 功耗

如果你不打算用國產毒瘤軟件就不用看了...

1.原理部分

我們先理清楚一個思路：

整機功耗=CPU 功耗 + 顯卡功耗 + 屏幕功耗 + 其它功耗

爲什麼其它功耗被稱爲其它功耗——與前三者相比實在不值一提

怎麼控制顯卡功耗呢？如果你用核顯那就沒意義了，如果你用英偉達，那有 Maxwell bios tweaker 和 Nvidia Inspector，在可以寫 vbios 的卡里一般調節 P08 來降壓，不可寫的用 Inspector 稍微限制下頻率。如果你用農企，那還談什麼功耗。所以剛纔都是我口胡不用看了。

怎麼控制屏幕功耗呢？亮，功耗大，暗，功耗小，如果說還有什麼，就是分辨率也是有影響的，這個很重要。

當然這些都不是正題，我們再建立一個邏輯：

同一顆 CPU，頻率越高，性能越強，功耗越大。

頻率和性能是線性關係，頻率性能和功耗卻是非線性關係。

這是因爲提升頻率，需要加電壓，加電壓會增加功耗，而且不是線性。

打開 AIDA64，計算機——傳感器選項，拖到最下面。

Downloads | AIDA64

CPU Package 就可以認爲是 CPU 總功耗，分四部分，其中 IA Cores 是 CPU 部分的功耗，GT Cores 是核顯部分的功耗，這兩塊是大頭。

intel 給所有 CPU 都設定了一個上限，叫做 TDP（散熱設計功耗），對於低壓 CPU 而言，這個值通常爲 15W/17W。

按 intel 的說法，就是您的機器的散熱能力，足夠支持 15W 的 CPU，沒別的意思了。

按用戶的說法，就是如果您讓機器滿載運行，假設屏幕耗電是 4W，您的機器沒有獨顯，電池是 80Wh，那麼通常結果就是 4 小時沒電。

TDP=15W 的意義，就在於長時間運行的情況下，它會把 CPU Package 控制在 15W 以下。

打個比方：

intel 讓一顆即將出廠的 CPU 做運算，同時一點一點加電壓，然後它頻率開始上升，運算速度越來越快，功耗也開始上升，當上升到 15W 的時候，停，瞥一眼頻率，2.7Ghz，還行，標上 i5-6200U。

intel 讓一顆即將出廠的 CPU 做運算，同時一點一點加電壓，然後它頻率開始上升，運算速度越來越快，功耗也開始上升，當上升到 15W 的時候，停，瞥一眼頻率，挖槽，3.2Ghz，標上 i7-6600U。

intel 讓一顆即將出廠的 CPU 做運算，同時一點一點加電壓，然後它頻率開始上升，運算速度越來越快，功耗也開始上升，當上升到 7W 的時候，停，瞥一眼頻率，2.0Ghz，標上 m3-6y30。

2.Intel® Extreme Tuning Utility (Intel® XTU)

XTU 是 intel 官方出品的 CPU 控制工具，以往是至尊 U 專享。這次在 Skylake 平臺上開放程度比較高，很多 CPU 都可以自行調節。

Support for Intel® Extreme Tuning Utility (Intel® XTU)

Turbo Boost Power Max，可以理解爲上面的 TDP

Core Voltage Offset，就是加的電壓

Turbo Boost Short Power Max，可以理解爲短時間運行情況下的 TDP

Turbo Boost Power Time Window，可以理解爲短時間運行究竟是幾秒

假設您使用了一顆 i7-6600u

您會發現，Turbo Boost Power Max=15W，Turbo Boost Short Power Max=25W，Turbo Boost Power Time Window=28s。

如果您在 XTU 裏給它改成 Turbo Boost Power Max=4.5W，Turbo Boost Short Power Max=7W，Turbo Boost Power Time Window=28s，然後點擊一下 Apply，恭喜您得到了一顆 m7-6y75。

但您突然打算開把 dota，這時候再打開 XTU 改回去嗎，這就太麻煩了。

把這個下載下來，解壓到 C 盤根目錄，然後打開 C:/PowerManager，運行裏面的.reg 文件。

回到桌面，點擊右鍵：

感謝教主 @可可蘇瑪 提供的指導

滿血，它就是 i7。平衡，它就是 corem。節能，它就是 atom。

個性化設置：

打開 C:/PowerManager 文件夾下的.ps1 文件，找到如下內容，修改 -v 後面的數值即可：

& 'C:\Program Files (x86)\Intel\Intel(R) Extreme Tuning Utility\Client\XTUCli.exe' -t -id 34 -v -67.3828125

&'C:\Program Files (x86)\Intel\Intel(R) Extreme Tuning Utility\Client\XTUCli.exe' -t -id 48 -v 4.500

&'C:\Program Files (x86)\Intel\Intel(R) Extreme Tuning Utility\Client\XTUCli.exe' -t -id 47 -v 4.500

&'C:\Program Files (x86)\Intel\Intel(R) Extreme Tuning Utility\Client\XTUCli.exe' -t -id 66 -v 32

第一排是電壓設置，-67.3828125 是降壓 0.067V，因爲 intel 電壓會給的比較保守，實際上再降一點，可以在相同功耗下跑到更高的頻率。願意作死的，可以降個 0.1V 以上看看。

第二排是 TDP 設置。

第三排是短時運行允許 TDP 設置。

最後一排是短時運行允許時間。
