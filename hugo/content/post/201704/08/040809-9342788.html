一句話概括的話，大數據是原料，雲計算是容器，商業智能是工具。

要詳細剖析的話，我們可以將這三個概念兩兩組合起來看它們之間的關係。

大數據 和 雲計算

雲計算的進步，是大數據產業擴張的一個重要基礎。

我們都知道，有一個概念叫“數據重力”，大致是說，所有待分析的數據都要集中到一個地方，然後才能進行分析。但是，傳統的本地數據中心能容納的數據量始終有限。

雲計算的出現打破了這重侷限性。雲可以把衆多數據聚集起來。這樣一來，雲上的數據集就是最大的，那麼當然，雲端就成了數據分析最理想的地點。換句話說，要充分利用大數據的「大」，就要靠雲技術來實現。

大數據和雲計算的發展前景，其實已經很明確了——大數據會往雲端遷徙。這是因爲，在“雲”上產生的數據越來越多地超過了在本地產生的數據。由“數據重力”的概念可知，數據的重心在雲端，所以數據整體都會往雲端移動。

根據商業智能公司 AtScale 的一項調查，72%的人計劃在雲端進行大數據工作。這個數字勢必越來越大。

舉例來說，亞馬遜 AWS 雲服務就支持了很多公司的雲端項目，包括飛利浦的醫療保健部門的雲端數碼平臺（Philips HealthSuite Digital Platform ）。這個平臺能分析和存儲 15PB 的患者數據，拓展性、表現力和可靠性都很高。（https://aws.amazon.com/cn/solutions/case-studies/philips/）

所以“雲”作爲一個海納百川的數據中心，能激發大數據的真正潛力——提高效率，降低成本，拓展格局。

大數據 和 商業智能

這兩者之間的關係就更好解釋了。正如之前說的，大數據是原材料，商業智能是工具。實際上，商業智能就是實現大數據價值的一個重要通道。

大數據應用的難點在於量大且複雜，而且非結構性數據佔了其中的絕大多數。在數據這行乾的人都知道，搞定非結構性數據、清洗數據等等這些工作，都是非常消耗人力物力資源的。

商業智能的主要工作就是幫你搞定數據——快速、及時和有效地清洗、整合、分析以及可視化數據，迅速地將數據轉換成圖表信息。這樣一來，決策人能夠很快了解信息作出決策，採取行動，及其高效地實現商業價值。

大數據發展勢頭越來越猛，所有人都在想辦法利用它的價值，而商業智能就是最受歡迎的方式。祭出一張用過數次的圖：

我們可以看到，出於對效率、成本和有效性的綜合考慮，受訪者當中 75% 考慮通過 BI（商業智能）來運用大數據，獲取大數據中的有效信息。

至於前景，我們也可以從 AtScale 的調查中發現，商業智能正在成爲越來越多人的選擇。關於商業智能領域的發展，這個在 Gartner 每年的魔力象限報告中，也都會提到領域內人士對行業發展的看法，我個人認爲是比較權威的：2017 商業智能 BI 與分析魔力象限 - 總結以及如何正確地閱讀 - 知乎專欄

雲計算 和 商業智能

到這裏，想必你已經很清楚，雲端商業智能一定是未來的發展前景。其中的道理和前面所敘述的一致——數據在雲端，分析當然也在雲端，那商業智能自然基於雲端是最好的。

依靠雲計算的雲端商業智能，與傳統商業智能比起來，就擁有了以下幾點優勢：

1. 高效。

商業智能分析流程的自動化和高效是毋庸置疑的。傳統模式的 BI 可能要花好幾個月才能得到的結果，SaaS 雲技術的 BI 分析速度增快很多，甚至可以說是實時的。

2. 靈活。

傳統的 BI 需要業務人員在開始分析前，與 IT 技術人員事先定好分析結構，在數據做出來之後，想要修改就要再去與 IT 人員商量重做。而云端 BI，只需要業務人員自己在後臺進行拖拽操作調整結構，就能馬上靈活得到新的數據分析。

3. 可靠。

使用雲端 BI 不必操心繁瑣的基礎設備管理、拓展性能、系統榮譽、平臺管控等等，也不用操心軟件更新——雲 BI 是隨時更新的，而且迭代非常快。

4. 安全。

很多人覺得雲端不安全，因爲數據脫離了自己的物理掌控。實際上，雲端比本地更安全——一箇中型企業對 BI 安全系統的投資，顯然很難跟專門從事這個領域的巨頭亞馬遜 AWS 相比。所以，將數據存在像 AWS 這樣的公司，就跟請了強大的安保團隊差不多，比鎖在自己不牢靠的抽屜裏要安全多了。

5. 低成本。

雲 BI 的低成本也是有目共睹的——不需要硬件投入，初期投入也較低。同時，因爲迭代快、沒有後續升級費用，性價比也比傳統 BI 更高。

綜上所述，要實現大數據真正的價值，雲計算和商業智能缺一不可。而云和商業智能的結合，更是搞定大數據的利器。可以說，這三者的組合會成爲未來商業決策的標準模式。
