微信在 2016 年 9 月 21 日發佈的小程序內測版，今天（2016 年 11 月 3 日）發佈了公測版，我司拿到了內測資格，在這段時間我們完成了微信小程序的開發，中間遇到了一些問題，也有一些思考，下面我會從微信支付、公衆平臺的迭代、微信生態系統、Open Web 和開發者這幾個角度來分析一下微信小程序。首先看一下什麼是小程序。

小程序是一種不需要下載安裝即可使用的應用，它實現了應用“觸手可及”的夢想，用戶掃一掃或者搜一下即可打開應用。也體現了“用完即走”的理念，用戶不用關心是否安裝太多應用的問題。應用將無處不在，隨時可用，但又無需安裝卸載。 - Allen Zhang

想要理解小程序，不得不提一下神的演講。在 2016 年 1 月 11 日的微信公開課 PRO 版現場，張小龍獻出了他的首次公開演講，闡述了自己對於微信的設想，還講述了微信的四個價值觀。

  • 一切以用戶價值爲依歸：大家都明白用戶很重要，但真正把用戶價值第一做到產品裏面去的不多，大部分只是把這個作爲一句口頭禪在說，但是在微信的平臺裏面，我們把這個作爲第一要事，作爲最重要的一個因素。
  • 讓創造發揮價值：公衆平臺從它誕生的第一天起，他的目標是要讓真正有價值的東西發揮出它的價值。我們是希望讓這個平臺裏面涌現出更多的有創造力的事情出來，而不是說這個平臺就是一個做流量的地方。
  • 好的產品應該是用完即走的：我們要考慮的則是怎麼樣更高效率幫助用戶完成任務，而不是讓用戶在微信裏面永遠都有處理不完的事情。
  • 讓商業化存在於無形之中：我們希望微信能做很好的商業化，但是它不是基於騷擾的、基於流量變現的商業化。我們的本意並不是要做成一個只是傳播內容的平臺，我們一直說我們是要做一個提供服務的平臺。我們希望存在一種新的公衆號的形態，這種形態下面用戶關注了一個公衆號，就像安裝了一個 APP 一樣，這樣的話我們可以嘗試做到讓更多的 APP 有一種更輕量的形態，但是又更好使用的一種形態來存在，這是我們在探討的一種新的公衆號形態，叫應用號。

連接人與服務

一說起微信，大家就會想到搶紅包，微信經過幾個大的版本迭代和幾次大的活動推廣，幾乎所有用戶都會搶紅包了，大部分用戶都已經綁定了銀行卡。也就是說，支付這條路已經鋪好了，可以沿街開店了，小程序應勢而出！

回顧一下互聯網過去的 20 年，不管科技處如何進步，人們內心深處的需求從來沒有變過，只是科技讓這種需求變得觸手可及：

  • 人和信息的連接，門戶新聞搜索引擎，有雅虎、Google
  • 人和人的連接，有 QQ、微博、陌陌
  • 人和商品的連接，有淘寶、京東、亞馬遜
  • 人和服務的連接，有微票、滴滴 Uber、Airbnb

當下火熱的網紅模式，大家再熟悉不過了，國內發達的網紅生態鏈作爲具有中國特色的致富方式讓國內外人士驚歎不已，甚至 BBC 都調查出了中國大陸上遍佈着 50 多家網紅孵化器（從零培養一名網紅，創造出一條“保姆 + 經紀人 + 供應鏈”的模式）。其實說白了，網紅就是流量加商業變現，在微博上刷流量，引導去淘寶消費。都是阿里系的，配合的很好，正因此，新浪微博迎來了第二春。網紅模式主要是包含了上面的第二點和第三點，連接人和連接商品。

而微信把這些全部連接起來，信息、人、商品和服務，形成一個生態閉環，所有的一切都在微信內部，以最少路徑最少路徑最少路徑完成用戶的需求，因爲微信支付這條康莊大道已經修好了。這就是 內容 + 連接 + 推廣 + 變現，這些生態環節的生產成本，生產內容的成本、建立連接的成本、推廣成本和支付成本全部降低時，一切都簡單了。

過去的十年，互聯網解決了人與信息、商品和人的鏈接，未來的十年一定是鏈接人與服務。微信發展到小程序這個階段應該說是有其背後的野心的。只是這個野心一直都裝在張小龍的心裏，保護得很好。

小程序誕生記

在微信誕生伊始，爲了豐富內部服務，嘗試開發了一些小插件，比如羣發助手，但是並不是很流行。然後出現了功能相對聚焦的訂閱號和服務號，旨在爲用戶提供信息資訊和服務，訂閱號可以說非常成功，服務號表現差一點，也比較成功，但是這兩個產品可定製化程度不高，二次開發也比較難，難以滿足千萬商戶和億萬用戶的需求。但作爲產品迭代升級中的一環，這是承前啓後的一環。

有大部分用戶也注意到，微信錢包裏面有很多第三方 Web App：滴滴、機票、京東、美麗說、電影演出票。在這裏也能獲得完整的 APP 的體驗，但有兩點不好的地方：用戶體驗不如原生 APP 好，這些服務都是騰訊投資的公司，用戶沒有可選擇性，只能使用這幾個服務。

現在，小程序誕生了，作爲以上產品和服務的一個迭代，比小插件功能更強大，比訂閱號服務號自由度更高，比 Web App 體驗更好。很多人現在把微信比作 OS，那麼小程序就是微信 OS 的第一個產品，這只是中點，並不是終點。可以預見，平穩發展很久的微信，已經準備二次發力了。

生態系統

我們希望基於微信能夠搭建一個生態系統，但不是我們自己把生態系統裏面的每一部分內容都做了。簡單地說，我們希望建造一個森林，而不是建造一座自己的宮殿；我們希望培育一個環境，讓所有的動植物在森林裏自由生長出來，而不是由我們建造出來。 - Allen Zhang

所以微信就是大自然，公衆平臺（訂閱號、服務號、小程序都屬於公衆平臺）這個平臺好比土地，微信支付好比空氣和水，而數以萬計的公衆號和小程序，就是大自然中的動物和植物。優勝劣汰，適者生存，微信是不會在乎個體的生存和死亡的，反而願意看到他們殘酷的競爭。因爲只有這樣，經過野蠻生長，經過廝殺，經過春夏秋冬，存活下來的，纔是微信用戶真正需要的產品，微信用戶纔得到了好的服務，這纔是微信願意看到的。

很多人猜測微信會提供一個小程序分發平臺，有這些猜想的同學，一看就沒有好好閱讀微信公開課 PRO 上神的演講。

微信要實現一個去中心化的動態系統，不會提供一箇中心化的流量入口來給所有的公衆平臺方、第三方。

沒有榜單，就沒有刷榜（好耳熟的樣子），那麼中小團隊就可以長舒一口氣了。而且，移動 App 分發時代已經結束，君不見，百度當年以 19 億美元收購 91 無線，以爲拿到了移動世界的一張船票，結果發現只是 91 無線只是一艘隨着大浪舞動的小船，現如今準備出售 91 無線下 iOS 業務，價格還不知道，但可以參考豌豆莢，錯過頂峯時期阿里巴巴 15 億美元的橄欖枝，三年後的今天以 2 億美元賣身阿里巴巴。

Open Web

在知乎上幾篇關於 Web 的討論很精彩，很多人認爲微信不夠 Web，不夠開放。

撇去技術細節，看一下國內的現狀，Web 這麼年多了，公司組織個人，擁有 Web 站點或者博客的並不多，而且用戶體驗也不太好。再看一下 2011 年誕生的微信，擁有微信公衆號的組織個人應該不在少數，其便捷性和實用性，較之 Web 是高出不少的。幾乎所有微信用戶都關注過或者使用過微信公衆號，然而這些人甚至不曾打開過一個 Web 站點。小程序上線以後，Web 的用戶體驗更是無法與小程序抗衡。相對來說，這是一種封閉，如果所有的商業寡頭都執行這樣的封閉策略的話，對於開發者來說不是一個好消息，因爲就沒有標準化這一說了，瀏覽器生態將會被架空，繼而會影響到普通用戶的體驗以及互聯網的長遠發展。

開發者

Native App 和 Web App 爭論了很久，就不多說了，小程序的用戶體驗比 Web App 好很多（真機實測），相對於 Native App，還是有一點差距，這是從性能方面來說，就整體體驗而言，小程序是高出 Native App 的，因爲微信提供了 UI 框架並強調各個小程序與微信的體驗一致性，用戶能比較快適應，同時還是用微信的賬號體系，減少登陸註冊環節，在加上小程序觸手可及的特點，這種體驗比用 Native App 會舒服一些。

下面說一下開發流程和開發體驗，公測有一定門檻，限制了註冊範圍：企業、政府、媒體和其他組織（個人開發者待定）。這裏有必要提醒一下開發者，拿到開放公測了，記得先申請微信認證，然後開通微信支付權限，小程序的支付和公衆號的支付是分開的，我司因爲申請的比較晚，整個國慶期間都是正在審覈中，不能調試微信支付。

小程序使用類 HTML5 技術開發，還擁有豐富的控件和系統調用能力，但他不是 Web App，也不是 Native App，而是兩者的一種中間狀態，開發完成後，需要提交微信審覈（包大小限制最大是 1M），其實小程序還是需要下載的，用戶在第一次打開的時候，還是需要加載審覈過的這個包，不過很小，下載這個過程可以忽略。同時小程序沒有外鏈，不支持 URL 跳轉， 也不能跳轉到 APP。

微信提供了一個簡單的開發框架，目標是通過儘可能簡單、高效的方式讓開發者可以在微信中開發具有原生 APP 體驗的服務。框架擁有自己的視圖層描述語言 WXML 和 WXSS，以及基於 JavaScript 的邏輯層框架，並在視圖層與邏輯層間提供了數據傳輸和事件系統，可以讓開發者可以方便的聚焦於數據與邏輯上。

WXML 借鑑了 React 的組件的思路。WXSS 類似 React Native 中的 CSS flex（其實是 CSS 的一個子集），但是微信還對他進行了擴展。你應該看出來了，這就是 HTML5 所涵蓋的技能棧。而且還能看出來，這大量借鑑了 React 和 Vuejs 的思路。

這是我之前做 React & Redux 的分享時寫的 Keynote，主要講的是在前端開發中，實現 UI 層和邏輯層的分離，這裏邏輯層主要基於 Redux（但這並不是必須的，邏輯層只要能處理好業務邏輯、輸出數據就好了），然後上層 UI 層可以是任何能夠構建 UI 的框架，基於此，能夠實現很多客戶端應用的開發，聰明如你，一定發現了，小程序也是這種做法！作爲前端開發者，框架很多，並不是非此即彼，應該多看看，現在如果還不瞭解 React 或者 Vuejs 的套路的話，很容易落伍的。若是瞭解這些，小程序上手輕而易舉！

一些八卦

  • 蘋果不讓叫應用號，所以改叫小程序
  • 不能做遊戲、直播產品
  • 一個用戶最多隻能添加 20 個小程序
  • 會和微信搜索打通，這個功能將會很強大
  • 好像沒有多少人關注“朋友圈本地廣告正式上線”這條新聞了

以微信在 Apple 發佈會上出現的次數，很多事情應該是和 Apple 打過照面了，都是商業公司，微信和小程序被下架的可能性基本沒有，不過和 Apple 之間肯定存在很多較量和制衡。

哪些服務和企業需要小程序

小程序開放後，肯定會有一大波服務號會利用小程序做一些嘗試的。絕大部分創業公司，可以使用這樣的配置，一個訂閱號（吸粉和推廣），一個小程序，就夠了，各種低門檻低成本，快速迭代驗證，一上來就三板斧很容易死掉。有把握了以後，根據產品形態和團隊情況，有選擇性的轉入 React Native、Web App、Android、iOS，初期的訂閱號和小程序不能廢了。大公司也會參與進來的，滴滴、美團、京東，去哪兒、大衆點評，別問我怎麼知道，這還要問嗎，所有人遲早都會被捲進來的。好好做小程序，別老想着微信會弄死你，只要不誘導分享，污染用戶的朋友圈，不搗亂，產品服務好，微信怎麼捨得弄你呢。

其實也並不是說小程序有多厲害，和他類似的產品有很多，百度直達號，Chrome App，都不溫不火，之所以看好小程序，主要還是微信這個平臺太生猛，因爲用戶都在這裏。萬事俱備，只需要你有很好的產品方向以及執行力。

但行好事，莫問前程。
