互聯網公司在面試數據分析師的工作時具體會被問什麼樣的問題？

我是金融科班出身，本科一畢業就去 BAT 做了數據分析師，目前也是面試官之一。

我被面試，和麪試別人的時候，主要關注 7 個方面。

1. 基本工具

包括規定動作和自選動作兩類。目前我所在的組不需要關心數據來源和結構化的問題，有專門的工程師團隊寫爬蟲、做清洗、維護計算集羣和數據庫。所以主要考察點在於查詢和衍生指標的計算方面。

1.1 規定動作

SQL 查詢：JOIN ON、DISTINCT、GROUP BY、ORDER BY 等等。從數據庫中提取數據是數據分析的第一步。另外我們的數據規模是 TB 級的，所以還要能使用 SQL 讓集羣做一些簡單的計算，不然都下載到本地的話運算資源是肯定不夠的。可能還會問一些非常基礎的問題，比如 PRIMARY KEY、int、str、double 之類。

Excel：數據透視表、VLOOKUP、COUNTIFS、SUMIFS、VAR.P、條件格式等等，可能會涉及到諸如 VLOOKUP 中的 TRUE 和 FALSE 參數有什麼區別，VAR.P 和 VAR.S 有什麼區別等細節問題。

1.2 自選動作

根據簡歷來問，簡歷上寫什麼就問什麼，會問得比較深入。Python、Stata、R、SPSS、SAS、EViews 都算比較常見的數據分析工具。順便奉勸各位不要作死，畢竟不作死都有可能會死。比如簡歷上寫“精通 Python”，雖然我知道簡歷注水是常態，但既然都“精通”了，那我問到 pandas，regular expression，DataFrame.iterrows() 返回的是 Series 還是 dictionary 還是 list of tuples，tuple 和 list 的區別的時候好歹都得答出來吧……

2. 邏輯思維

主要分爲兩方面，對業務邏輯的理解能力和行文的邏輯水平。

2.1 業務邏輯

雖然一個業務看似流程簡單清晰，但產生數據的複雜程度往往超過大多數人的想象。對業務邏輯的考察主要通過相關項目經歷。如果是典型的學校項目，我會比較關心指標設計選取、代理變量選擇、誤差分析、因果性解釋等。

這裏再次奉勸各位不要作死，寫在簡歷上的項目經歷起碼自己要非常熟悉，對答如流。如果我聽你介紹 15 秒項目後提出的問題（如“你爲什麼說 北京經濟適用房建築面積與房屋建造年份的乘積 是一個非常重要且有實際意義的解釋變量？”）就能把你難住的話，那你也會把我難住的——尼瑪面試評價表怎麼寫啊摔！

以我爲例，我每天接觸的是 700 多張表，每張表的字段往往超過 200 個。這些表和字段往往還有關聯。面對這麼多業務指標，能否迅速理解它們之間的聯繫? 面對新的數據需求，能否邏輯清晰地將它拆分成指標、二級指標並進行各種計算? 面對複雜的局部最優化和全局最優化需求，能否“抓大放小”，能否迅速找到關鍵控制點、關鍵影響因素並加以優化? 我每天通常只有不到 1 小時的時間用來出一份要發給 CEO 等大佬的，關於業務數據分析和後續指導意見的報告。思維不敏捷，邏輯不清晰的話，是很難做好這份工作的。

2.2 行文邏輯

畢竟最終產出是一份份報告，可能是 HTML 郵件也能是 PDF。文章結構還是很重要的。這裏不展開說了，不過關鍵的幾點是先說結論，先寫摘要。

3. 理論儲備

也分爲規定動作和可選動作。

3.1 規定動作

主要是基礎的統計學理論，如方差、協方差、算數平均數、幾何平均數、中位數、衆數、分位值、雙峯數據、長尾數據、假設檢驗、期望迭代法則、貝葉斯原理等。

3.2 自選動作

根據簡歷來問，簡歷上寫什麼我一定會問什麼。第三次奉勸各位不要作死，寫的檢驗也好機器學習算法也好好歹自己要知道原理、適用條件、侷限性。不然我跟你聊起 Pearson distance、K-means cluster 的隨機性問題的時候你接不上來也是很尷尬的。

4. 對細節的敏感度

作爲數據分析師，每天要關注大量數據指標。對細節的敏感度是非常必要的。這主要分爲兩方面，對統計口徑的敏感度和對數據的敏感度。

4.1 統計口徑

統計口徑一致是確保數據可比性的基礎，這非常考驗數據分析師的敏感度和行業經驗。比如轉化率，是點擊算轉化還是註冊算轉化還是購買算轉化？配送時間，是從用戶下單開始計時還是從訂單確認開始計時還是從商品出庫開始計時？客單價包不包括配送費、打包費、代金券形式的折扣優惠？

4.2 數據

我非常關心候選人對數據異常波動、離羣值、平均數沒有代表意義等情況的迅速識別能力。比如已知然壽司套餐單價 1,500，酒水單價 300，平均客單價 2,500，能不能馬上想到這可能是雙峯數據或者長尾數據，抑或既雙峯又長尾的數據？

5. 學習能力

互聯網行業瞬息萬變，光數據的存儲就有 Oracle、MySQL、Hadoop、Spark、Hive、Impala、谷哥哥三駕馬車等一大堆奇奇怪怪的東西。互聯網行業的從業者經常要面對新需求、新工具、新方法。能否迅速掌握新知識，解決新問題是候選人必須證明給我看的。

主要考察的方式是瞭解過往項目經歷，或者我出作業題（比如 Sci-Hub）。

6. 排版和簡單 UI 設計

我認爲數據分析報告必須簡潔、清晰、重點突出。主要考察方式是出作業題讓候選人限時交一份 slides（就是 PPT 啦）出來。能掌握標準的 Microsoft Design Language 是大大的加分項。

7. 價值觀

主要看工作熱情、態度、道德水平等等，這方面我問的問題比較隨機，沒什麼規律可循，甚至問過機械鍵盤、人體工程學設計等方面的問題。
