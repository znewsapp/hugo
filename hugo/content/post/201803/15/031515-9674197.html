對一線大廠而言，在技術面試的時候最看重的是什麼？

就我個人而言，首先我在篩簡歷的時候做以下一些點：

1. 如果寫了期望薪資，先看你的期望薪資。上限不怎麼設，如果下限太低會直接略過。如我之前在大搜車的時候，期望薪資在 10k 以下的會直接略過。不是說你能力不行，這個時候看的是你對自我能力的自信認知。況且，實際上 10k 真的不算什麼，估計評論下會有好多人嫌低，畢竟知乎大佬們。

2. 看你過往經歷。也就是看公司，廠子大小無所謂，我個人比較主觀，如果是一些耳熟能詳的小廠，我也會多留意。然後就是如果不耳熟能詳，但是公司名字或者看做的事情挺有意思的公司，我會自我擴充我的知識庫，去看看這是怎麼樣一個公司。畢竟除了 BAT 之外，各種 新晉獨角獸或者 geek 的小廠，都是極好的。

3. 如果有 GitHub 或者博客，我會與上一條同時關注，並視爲重要的第一印象。如果你的 GitHub 的 id 一眼看過去我以前在一些場合見過，會加分，沒有的話進去看如果有很多幹貨也是極好的，並擴充到我的個人大腦知識庫中。但是裏面要是沒什麼東西，還是建議不要留了，減分。

4. 看你的項目經驗，如果簡歷有寫的話。看看是否與團隊契合，是否有趣，是否能見真章。

5. 說實在的，我最不看的就是學校以及是本科還是專科還是碩士（需要強大學術背景的崗位除外）。你要是前面的都是好印象，這一條無關緊要，當然國際頂尖的學府也會多看一兩眼不過不會作爲標準，並不會因爲是什麼牛津，麻省理工什麼的就會放寬前面的要求，講真這些學校的被刷的一樣有，不過按比例來說我纔會少些。並且就我個人而言，不會因爲學校刷人。

在面試的時候，由於水平太低，所以只有一面過。拿老東家舉例，一到三年的人我會注意以下幾點：

1. 問之前做的事，並且看看能否講清楚。有些人只是在項目中做一部分，這不要緊，有自己的主見並且會有規劃即可，能明明白白知道自己在做什麼，並且做的事情在各方面與整個項目的關係，以及自己的一個定位。如果有獨立負責的項目，自然就要好好問一下了。

2. 問所在領域的基礎，比如 Node.js 方向，就是 Node.js 基礎以及數據庫、高可用、分佈式、簡單架構等等等等。當然也不是各方面要面面俱到，主要是看學習態度以及潛力。

3. 計算機科學基礎，一到三年的同學我必問，並且會輔以筆試。記得老東家 CTO 說得好，無論吹得多好，最後加入團隊之前還是要看看代碼寫得怎麼樣。很多人會反感面試的時候筆試，說什麼面向文檔編程。我在筆試的時候會問的是基礎算法和數據結構的問題，這是我認爲一個程序員必須要掌握的，哪怕在工作中用得不多。如果你被我面試過，應該會有印象，而且這是我每次面試必備的。當然，也有可能會是一些語言方面的題目，這個時候你要是說 API 記得不是很牢，沒關係，可以在相應的地方跟我說一下你是怎麼想的，畢竟我看的是你的思路清晰不清晰，代碼是否易懂或者思路正確。

4. 看你對技術的熱情，通過面試中附加一些問題得到。

以上僅爲我個人的面試習慣，非常有主觀色彩。每個面試官脾性不同。面試難度會隨着我對你的判斷而提高或者降低，如我覺得你的水平不止於簡單算法，那麼會增加難度。

最後，如果做法有誤，還望大家指正，如果對你有幫助，希望不要吝嗇你的贊。
