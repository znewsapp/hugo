整理了一下互聯網創業公司常用的黑話，供大家參考，可以避免理解上的偏差。每個段落都適用於不同的場景和不同的對象，大家猜猜看。

1. 老闆黑話

你來我辦公室一下 = 老子又想到了絕妙的 idea

得專注用戶體驗 = 界面畫的好看點

產品氣質不夠年輕 = 飽和度通通調最高

產品氣質不夠成熟 = 界面通通做成黑的

產品不夠大氣 = 我也不知道哪不好反正就是不好

要精緻的感覺 = 抄蘋果

要利用用戶的固有習慣 = 抄同行

要追求流行設計趨勢 = 抄微信 / 淘寶 / 滴滴…

你說的這是另一個問題=我說的纔是對的

你說的跟我是一個意思=我說的纔是對的

我們討論的是兩件事情=我說的纔是對的（感謝 @silence ）

我們是彈性工作制 = 加班不給錢

我們是扁平化管理 = 公司沒幾個人

我們會給你很多期權 = 不會給你很多工資

我們每天都有果盤！ = 可能是公司唯一的福利了

當務之急是搶佔市場 = 快狂發補貼

快速建立用戶羣體的壁壘 = 快拉他們進微信羣

要讓用戶產生自發傳播 = 快讓他們轉發朋友圈

爲了健康發展我們要啓動下輪融資 = 公司沒錢了

我們辭退了一些跟不上公司發展的同事 = 公司沒錢了

打補貼戰其實不符合我們公司的理念 = 公司沒錢了

2. 產品經理黑話

產品設計應該大道至簡 = 複雜的我也不會

用戶都應該用完即走 = 回不回來我不管

這是常規的典型的做法 = 我抄的微信 / 淘寶 / 滴滴…

需求要抓住人性 = 多放美女照片

得考慮用戶的使用場景 = 加個夜間模式

商業模式要形成閉環 = 放東西線上賣

要搭建完善的用戶運營體系 = 做個積分商城

要有社交元素促進活躍度 = 塞一個 IM

沒有用戶是因爲沒有做好運營 = 不是我的鍋

體驗不好是因爲技術實現問題 = 不是我的鍋

界面難看是因爲設計水平不行 = 反正都不是我的鍋

這個我回去再確認一下 = 別說了，老子沒想到（感謝 @BenWang ）

在嗎？= 要改需求了

哥哥最近累嗎要不要喝奶茶 = 要改需求了

那個…有句話…不知… = 要改需求了

下次肯定不改了 = 這次先改了再說

你的建議很好我們已經想到了 = 我擦說得真有道理趕緊提需求

你的建議我們會考慮的 = 這建議好像有點蠢

你的建議很有啓發性 = 哈哈哈什麼傑寶玩意兒

3. 程序員黑話

你這個需求不清晰 = 我不想做

目前技術實現不了 = 我不想做

現在服務器性能跟不上 = 我不想做

去找老闆 / 項目經理排期吧 = 我不想做（感謝 @矮米粒 ）

你怎麼老改需求啊 = X 了個 X

你怎麼設計的這麼複雜啊 = X 了個 X

你怎麼就給這麼短時間啊 = X 了個 X

在嗎？ = 要延期了

弟弟最近累嗎要不要喝奶茶 = 要延期了

那個…有句話…不知… = 要延期了

下次肯定不延期了 = 這次先應付了再說

你退後點說話 = 口水別噴到我寶貝鍵盤上

你別把胸壓到桌子上 = 我寶貝鍵盤會受不了的

你告訴我輸什麼我自己來 = 你別用髒手碰我的寶貝鍵盤

你怎麼還用 Word 啊？ = Markdown 纔是最好的寫作工具

你怎麼還在用 ThinkPad 啊？ = Mac 纔是最好的電腦

你怎麼還在自學 Python 啊？ = PHP 纔是最好的語言

感覺還不是很全。歡迎補充。

我們在尋找合夥人 = 就差一個程序員了
