回答來自機構帳號：少數派

北京時間 2017 年 3 月 23 日星期四，蘋果對 Workflow 完成了收購。根據 TechCrunch 的 報道，交易已經結束，具體金額目前未知。

收購的對象包括 Workflow 這款 iOS 應用，以及它背後的技術與四位開發者組成的團隊：Ari Weinstein、Conrad Kramer、Ayaka Nonaka 和 Nick Frey。

（圖爲 2015 年 Workflow 獲得了 Apple Design Award 的照片，當時另一位開發者 Ayaka Nonaka 還未加入）

被收購的 Workflow 將繼續保持開發，今後仍可以在 App Store 下載，並且價格變爲免費。

Workflow 是什麼

Workflow 是 2014 年 12 月發佈的一款 iOS 應用。和一般的應用不大一樣，它本身沒有什麼功能，但你可以用它提供的動作模塊來像搭積木一樣組成各種工具，滿足自己的需求：

簡單一點的，你可以讓 Workflow 根據你的所在地，隨機地選擇一家你周圍的飯店，解決吃飯難的問題；難一點的，你可以把微博裏的秒拍視頻抓到自己的相冊裏，直接分享給好友；在 Apple Music 聽到喜歡的歌，想配圖分享出來時，你也可以做一個 Workflow 抓取那首歌的高清封面；經常寫作的人可以用 Workflow 把截圖上傳到你的所用圖牀；可以把閱讀時遇到的生單詞加到正在背的扇貝單詞庫中；可以把視頻轉成 GIF；也可以反過來把 GIF 轉成視頻；可以用它以圖識圖；可以用它識別手機截圖裏的二維碼……

再往下我還可以說很多 Workflow 能做的事，但相信你已經感受到了它的能力。

你可能會想，這樣一款應用是不是上手很難？那你就錯了。Workflow 的功能雖然不簡單，但也稱不上難，我稱之爲「跳一下就夠得到的高度」。如果願意學，上手它很容易，學習的坡度也比較緩。

如果你想對 Workflow 有更深入的瞭解，可以看我之前寫過的系列教程：《從零開始用 Workflow》

蘋果看上了 Workflow 什麼？

蘋果之前也收購過在 App Store 上架了應用的公司，要麼是收人，要麼是收技術，一般不會把應用留下。但 Workflow 這次把應用本身、技術和團隊打包收購，足以看出蘋果對 Workflow 這款應用的重視，或者說，是對 Workflow 這款應用目前實現方式的認可。

對於多數人來說，Workflow 最直觀的優秀之處有兩個：一是它形式上突破了 iOS 的沙盒限制，你可以從一個應用獲取信息，發送到另一個應用中，比如你可以在豆瓣電影裏不離開 App，然後按照一定的格式 把電影加到 Todoist 中；二是它操作要比同類應用更加容易，另一款自動化領域的明星應用 Pythonista 的實際能力要在 Workflow 之上，評價也很高。但因爲使用它要用上實打實的編程語言，門檻陡增。Workflow 不光有人性化的圖形界面和易於理解的拖拽式交互，它還內置了許多動作，可以省下使用者用來思考或操作的時間。比如你可以用 Apple 的 API 獲取 iTunes Store 和 App Store 裏商品的信息，但這個動作做出來會偏複雜。於是 Workflow 就直接提供了一組動作，可以非常直觀地實現這個功能。

但這些都不是蘋果方面的發言，TechCrunch 的報道中，蘋果的發言是這樣的：

The Workflow app was selected for an Apple Design Award in 2015 because of its outstanding use of iOS accessibility features, in particular an outstanding implementation for VoiceOver with clearly labeled items, thoughtful hints, and drag/drop announcements, making the app usable and quickly accessible to those who are blind or low-vision.

看起來，蘋果似乎更認可的是 Workflow 爲視覺以及行動上有障礙的使用者提供了完善的 VoiceOver 服務。

就在 3 月 15 日，Workflow 的 Blog 上也更新出了 一則真實的故事，它講述了因爲大腦缺氧 45 分鐘，導致後天失明，幾乎失語，外加行動有嚴重障礙的 Logan 在日常生活中是如何使用 Workflow 讓智能語音設備 Echo 告訴他天氣的。除此之外，Logan 還會用 Workflow 給朋友發短信。娛樂方面，Logan 能用 Workflow 控制播放器，他可以播放、暫停音樂、切歌，甚至能在切歌之前，讓 Workflow 告訴他下一首歌是什麼。

The Logan Project - Workflow_騰訊視頻 http://v.qq.com/x/page/u0386tfsir0.html

當然，Workflow 只是一個輔助，真正值得讚揚的是 Logan 不屈服於命運的精神。但是毫無疑問，因爲 Workflow，Logan 在對抗命運這條路上有了強大的武器，讓他能更有自信。

Logan 所在的大學 Auburn University in Montgomery 現在成立了一個項目叫做 Logan Project，目的是爲了讓像 Logan 一樣身罹障礙的人士獲得更好和更有針對性的教育。

可以看出，蘋果對這種自身功能完善，並對殘障人士友好的作品是非常認可的。開發者們日後可以多往這個方向發力，多爲這些特殊人士考慮。我身在日本，發現日本的都市幾乎沒有地方是輪椅去不了的，幾乎所有地方都有電梯和緩坡。從某種程度上來說，這就是文明的象徵。開發者們相對會更開眼地看世界，能夠讓殘障人士更容易地使用你們的作品，就是在把世界變得更好。

樂觀預估

Workflow 這次被收購，瞬時出現了兩種聲音，樂觀者看見了希望，但科技界向來不缺乏悲觀論調和惡意調侃。我本人對這件事也有樂觀和悲觀兩方面的看法，在此打算先寫一下可能發生的好事。

獲得更高權限，成全更多功能

被蘋果收購後，Workflow 獲取更多的權限幾乎是可以肯定的，我第一個希望它改善後臺常駐權限和通知中心的內存限制。後臺常駐權限或許能爲 Workflow 提供更多更靈活的運行方式，而通知中心的內存限制放開，則意味着 Workflow 可以在通知中心做之前不能做的事。

我另一個願望是讓 Workflow 支持 Siri。我有一個動作是通過 SSH 讓家裏的 iMac 進入睡眠，最便捷的方式是在 Apple Watch 上運行這個動作。如果 Workflow 支持了 Siri，我給 Apple Watch 說句話就能在被窩裏 1 讓 iMac 睡眠，全然不用任何複雜的方法，多爽。

多國本地化，對中文支持

蘋果的東西無論是軟件還是硬件，多國支持都非常出衆，尤其中國現在是它最重視的市場，Workflow 支持中文完全是可以預料的變化。

也許支持中文還是呼聲最高的變化。已經有太多人表示，如果 Workflow 支持中文，他們就用。我之前還嗆這種聲音，說：「你是想等全世界的東西都支持中文了再學，還是邊學語言邊學全世界的東西？」

現在看來這個論調雖然基本成立，但在 Workflow 這個例子上已經站不穩了。伸手黨們，你們贏了，我等着看你們在 Workflow 漢化後分享出驚人的動作。

Workflow for Mac 也許有希望

Workflow for Mac 的呼聲一直存在，因爲在 Mac 上的兩個近似物——Automator 和 Keyboard Maestro，要麼功能不夠強大，要麼強大得不好學，而且長得都不如 Workflow 可愛。

蘋果自家的 iOS 應用幾乎都在 macOS 上有對應的版本，而且對快捷鍵的支持都做得不錯，多窗口操作也更方便。

各位資深 Workflow 玩家，想象一下，有朝一日你能在 Mac 上編輯 Workflow，這是不是你夢寐以求的？

悲觀擔憂

不論是科技段子手的調侃，愛挑刺的人的慣性發揮，還是 Workflow 愛好者們的深切擔憂，大家出於對蘋果開放程度的懷疑，加之 Workflow 被收購同時發佈了一版「特殊的」更新，輿論中產生了另一種聲音。我本人也對這次的收購有一些不那麼樂觀的想法。

或許 Workflow 會不再那麼開放

也許大家還沒注意到，在昨天更新的 Workflow 1.7.3 版的 Changelog 裏，出現了這兩條內容：

  • Maps actions are now powered by Apple Maps
  • Get Street View Image, Google Chrome, Pocket, LINE, Telegram, Uber, and Workflow Gallery submissions are no longer supported

Workflow 的地圖功能現在僅由 Apple 提供，移除了谷歌街景預覽和 Chrome 的動作。也移除了對 Pocket、Line、Telegram、Uber 等服務的支持。同時，看起來是關閉了 Workflow 的分享區。

Pocket 等網絡服務被移除，有些人解讀是利益衝突的原因。Pocket 可以直接對應 Reading List，Line 和 Telegram 也可以對應 iMessages。但這樣不能解釋爲什麼 Instapaper 留下、Uber 爲什麼被移除。我個人對這些服務被移除比較樂觀，猜測是由於它們和 Workflow 的結合方式不符合蘋果規定或者什麼其它原因，也許是臨時的，日後還會回來。

但移除了對谷歌街景和 Chrome 的支持，看起來更像是一種蘋果對抗谷歌的防禦姿態。你可能會說，防禦姿態的話這次怎麼還支持了微軟的翻譯呢？那是因爲蘋果還沒出自家的翻譯啊！Workflow 之前可是用的谷歌翻譯。

這次更新還關閉了 Workflow 的分享區，這可是 Workflow 在 1.6 版本里着重推出的功能。看過 我寫的 Workflow 更新文章 的讀者應該都知道，Workflow 的百分位（1.x.x）更新都非常強悍，像對 HTTP 請求方式的支持、在 Workflow 運行另一個 Workflow 都是在百分位更新出的功能。Workflow 團隊把分享區放到十分位（1.x）來推，肯定是想要做好這個東西。所以不難推測，這次關閉分享區是蘋果權衡之後的決定。所以會不會再開以及什麼時候再開，就很難說了。

在收購之前，Workflow 是一羣天馬行空的年輕人的作品，他們大踏步向前跑，想到哪幹到哪，給 Workflow 接入了五花八門的服務和應用。但接下來，蘋果會從更多的角度來考慮這些服務或應用，是否適合繼續存在於 Workflow 之中。

完成度不夠與更新頻率慢

嚴格來說，目前 Workflow 離它應該做到的完成度仍有一段距離，最典型的一個問題是「整理難」。Workflow 至今尚未支持文件夾，不管你用什麼方法去分類你的 Workflow，這些 Workflow 都必然會平鋪在應用之中。

此外，Workflow 在拖拽動作時的表現一直都不穩定。被拖拽的動作會在移動的途中因爲莫名其妙的原因卡落在途中，而且還會改變那個 Workflow 卡片本身的顏色。

所以，如果你本身已經有很多 Workflow，然後又新做了一個，並要把它拖到比較靠前的位置，和它同類的 Workflow 匯合，你會發現在 iPhone 上是個漫長且充滿意外的旅程——它隨時可能頑皮地卡落在途中的任何位置，像變色龍一樣將自身的顏色和周圍的 Workflow 融爲一體，和你捉迷藏。

反正，我幾乎已經放棄了整理，只用搜索功能來找我的 Workflow。

除此之外，Workflow 還有不少其它小問題，比如在通知中心表現不穩定，還是會有動作跑着跑着卡死，再拉出通知中心後出現無法讀取的情況。

修復這些小問題，包括增加對更多應用和服務的支持，都需要短平快地更新，要開發者發現問題、解決問題、推出功能、發佈新版本。從今年三月開始往回倒，Workflow 平均每個月發佈一版更新，TestFlight 更是提前一到兩週發佈，從我加入 Workflow 的 TestFlight 以來，它每個版本的 TestFlight 至少要更新 3 次，因爲每次推出新功能就會有新 Bug 要修，所以可以說它的更新頻率非常高。

但是大家都知道，蘋果更新自家應用的節奏可不是這樣。蘋果喜歡搞大新聞，一年只會發佈一兩次比較醒目的更新。這種節奏和 Workflow 的步調很不一致。不過，從好的方面想，這會讓 Workflow 打磨得更精細，Workflow 確實也需要好好補補坑，打磨打磨細節。相信有了蘋果的品控，很多目前的小問題都將能夠變得舒服起來。

綜上，我希望蘋果能夠根據 Workflow 獨特的特性，在走得快和走得穩之間把握好平衡。

和 Automator 一樣被荒廢

這大概是目前可以想到的最悲觀的論調。因爲就在 4 個月前，蘋果剛剛撤除了 Sal Soghoian 的崗位。他曾在 04 年的 WWDC 上展示過 Automator，在那之後他一直負責通過腳本語言實現溝通系統和軟件之間的功能，以及 Automator。自 97 年加入蘋果到離開，他差不多爲蘋果工作了 20 年。

很多人認爲這個事件是蘋果不再重視用戶自動化（主要是 User Script & Automation）的信號。Sal Soghoian 本人似乎也有這種擔憂，他似乎覺得蘋果要在 App Extensions（類似 iOS 上的 Share Sheets）上發力，拋掉 User Automation。於是專門在 MacStories 寫了一篇《App Extensions Are Not a Replacement for User Automation》。

但從蘋果今天邁出的這一步可以猜測，蘋果或許是認爲在 Sal Soghoian 十多年的管理下，User Script 和 Automator 在相對更開放的 macOS 上也是雞肋一樣的存在。而 Workflow 用了短短兩年就在 iOS 上將自動化做得順風順水。

也許，蘋果只是想通過另一種方式，普通用戶更喜歡的方式，一種在未來甚至有可能淡化普通用戶和 Power User 之間那條線的方式，來實現系統內的自動化。

寫在最後

兩年前，在 Workflow 首發的一週後，我在第一篇關於 Workflow 的深度測評 末尾寫道：

我覺得這個應用的未來將會是一片光明。

現在看來，它是直接飛向太陽了。

回顧歷史，Workflow 真是堪稱傳奇。Ari Weinstein 越獄插件開發者的背景讓它有着比通常應用更受矚目的首發。它踩着 Launcher 的屍體發佈，實際上卻做着更出格的事 2 。儘管如此，Workflow 在發佈當日就獲得了 App Store 的編輯推薦，據我所知這樣的效率應用僅此一個。從此它一路高歌猛進，繼 URL Schemes 之後成爲 iOS Power User 的代名詞與必備技能。很少有應用像 Workflow 這樣每一次的小版本更新都值得認真對待，都值得專門寫一篇文章去講解。

Workflow 的出現本身也是一個有趣的故事。根據 Cult of Mac 的報道，最初這個團隊只有 Ari Weinstein 和 Conrad Kramer 兩人。Workflow 的初版原型只用了不到 48 個小時。

兩位開發者後來贏得了 Thiel 獎學金，這是專門爲 20 以下的年輕人提供的獎學金，獎金 10 萬美元，要求是讓他們暫停學業來創業或者直接進行科學研究。Weinstein 直接從麻省理工學院退學，Kramer 也從麻省理工學院休學，兩人開始了他們的創業生涯。

乍一看，這簡直是人生贏家的寫照，彷彿世界的一切地方都爲他們開啓了通行證，只要他們願意做他們喜歡做的事，後面就有人提供資助。但實際上，Weinstein 很早就在越獄圈有了名聲，他對編程、效率和自動化的熱愛持續了十幾年，並且在這條路上逐步深化纔有了這樣的成就。這讓我想到少數派有幾位高中生，他們本身非常優秀，已經習慣了在英文世界查找信息，動手能力也很強，做出了了不起的 Workflow 動作和樹莓派的各種使用方法。很多人在他們的文章下的留言是讚揚他們年輕有爲，在感嘆自己高中的時候自己幹了什麼。但是我想對這些孩子說，你們的人生纔剛開始，如果你們被這個小圈子裏的讚譽衝昏了頭腦，很有可能會葬送了本應更加光明的未來。

15 年末各個網站推出優秀應用榜單時我說過：如果當年哪個優秀應用榜單裏沒有 Workflow，那是這個榜單爛而不是 Workflow 爛。現在我依然是這個觀點。雖然有些過譽，但我覺得蘋果收購 Workflow，既是 Workflow 的福分，也是蘋果的福分。對於從小就在 iOS 系統上摸爬滾打的 Weinstein，獲得蘋果的認可自然是一種榮耀，但不可否認的是，蘋果現在也需要一款產品來挽救自動化方面的頹勢。

我相信 Workflow 團隊的成員們至此還沒有走到事業巔峯，我期待看見他們更輝煌的未來。於我而言，比起國內大佬們的心得或雞湯，他們正在書寫的人生才更值得注意和學習。

我也很期待 Workflow 的下一次更新，那將是蘋果接手之後的首秀，我們將在那次更新裏讀到更多蘋果對自動化方面的姿態。
