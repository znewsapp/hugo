在創業初期，如何管理好公司的財務賬目？

首先我們假定題主問的不是內控手段（創業公司也不適合太繁複的內控手冊，主要內控手段依靠員工口碑）

其次我們假定題主對財務系統的需求不只是報稅節稅。

那麼這個問題實質上是：一個可以支撐日後發展（pe 盡調，財務分析，ipo 審計等）的管理用財務系統，最簡單可以簡化到什麼程度？

簡單的說：可以簡化到只依靠會計賬 +Excel 臺賬就能達到 ipo 要求的程度，只要你做的足夠完整和細緻。

  1. 完整指的是會計賬：請記錄你所有的交易，如果你要做兩本帳，請把另一本帳也按照一個公司的標準做完整。這個 @Wifs P 大叔已經說的清楚了，不再敷言。
  2. 細緻指的是臺賬：請儘可能對每一個類別的交易做一個臺賬：

譬如你的採購，你的會計收到供應商的發票，請做個 excel 表格詳細記錄一下每一張發票對應交易的供應商名稱 / 產品 / 產品編碼 / 數量 / 金額 / 憑證號 / 入庫日期（月份）。

  • 這樣，當你的投資人問你的採購情況的時候，你可以拿出一張每個月採購量和採購單價的折現圖，來解釋爲什麼在產量大幅度提高的時候你的採購成本並沒有上升；
  • 這樣，當你的投資人質疑你的關聯方採購單價存在貓膩的時候，你可以拿出採購趨勢的量化比較和單價對比，並底氣十足地說：我說的每一個數字你都可以找到發票和銀行水單來對應。
  • 我知道即使你不做這些記錄，你也能記的你的經營情況並完整表述，但請相信我，如果沒有數據支撐，投資人 / 會計師能夠相信的，只有你的人品而已，你覺得投資人有多信你的人品怎麼樣？

譬如你的銷售，你的會計開具了銷售發票，請做個 Excel 表格詳細記錄一下每一張發票對於交易的客戶名稱 / 產品名稱 / 產品編號 / 類別 / 數量 / 金額 / 賬期 / 發生日期（月份）

  • 這樣，當你的投資人問你的銷售 / 產品結構，單價變化趨勢，主要客戶分佈，毛利分析以及憑什麼你覺得你的業務明年能擴大一倍的時候，你可以拿出一張產品結構 / 客戶分佈及單價趨勢折線圖告訴他們：過去三年我們每款產品 / 每個客戶的銷售週期大約在 2 年左右，今年我們新開發產品較去年同期增加了 2 倍，你聰明的，告訴我，明年我們收入預期是多少？
  • 同樣，你也可以用不容置疑的語氣說：我說的每一個數字，你都可以找到客戶簽收單和銀行水單來對應。
  • 我知道即使你不做這些記錄，你也能記的你的經營情況並完整表述，但請相信我，如果沒有數據支撐，投資人 / 會計師能夠相信的，只有你的人品而已，你覺得投資人有多信你的人品怎麼樣？

這樣的臺賬還有很多，以一個製造業公司爲例，還包括：收發存 / 生產報表 / 運輸 / 倉儲 / 工資 / 售後費用 / 在建工程歸集 / 固定資產等等臺賬。

核心的思想是：用你的會計帳爲主線，把整個公司的運營數據穿起來，所有臺賬的數據，都務必要和會計帳對應，數據保持一致。

你也許會說我爲什麼要讓會計蒐集分析這些運營數據，我大可以讓我的銷售 / 採購 / 費用管理 / 倉儲運輸部門做這些事情，然而有一天當你發現當你需要分析某項業務——譬如運輸費——的時候，你發現運輸部門的數據和財務部完全對不上：

如果你選擇相信運輸部門的數據，你可能會犯遺漏某個運輸事項的風險，畢竟只有財務部的支付 / 發票是最完整的（如果不是的話，請重讀一遍香川大叔的答案）；

如果你選擇相信財務部門的數據，你會發現你根本無法分析：因爲缺乏臺賬和運營數據，數據波動極易受到付款頻率的影響。

然而這並不是最可怕的部分——雖然我見過的大部分民營企業老闆，一方面覺得業務部門的數據不完整，一方面嫌棄財務數據的核算不準確的兩難境地——最可怕的部分在這裏：

你的運營部門可能流動性極高，兩年過去了，換了兩撥人，已經沒人找得到當年的業務數據了。最後我們只能依靠老會計的賬（通常財務部的流動性不高，且前後任交際工作做得比較好）來拼湊當年的業務數據，而這個時候券商警告你：IPO 快速過會的時間窗口很可能只有半年多，下一次政策放水誰也不知道是什麼時候了。

所以製作大量臺賬來確保財務數據的邏輯性和可讀性，通過會計帳來確保臺賬的完整性和良好保存是非常低成本且可靠有效的構築財務分析的基礎數據的手段。

說起來你可能不信，就算只是簡簡單單的這些工作，如果你堅持做下來，你可能已經超過了 95% 的創業公司了。

而 80% 以上的擬上市公司，都會因爲這些小問題拖 2 年以上的財務整改。

補充說明：

臺賬的細緻程度至少要達到業務部門分析使用的標準。譬如售後費用的分析，一開始可能只要把每筆售後業務加上客戶的名稱即可，後來質量部門覺得不夠，要在臺賬上加上不良品的型號，後來又覺得不夠，要加上損壞的部件類別；再後來人家回來又要對故障類別分爲嚴重 / 不嚴重兩類，甚至每筆售後服務上要附上故障分析報告或者報告的索引編號。

業務部門的需求經常在變，臺賬的細化是隨着公司業務能力一點點加強的。如果拘泥於某個財務軟件，那麼財務部門不可能和業務部門一起成長。
