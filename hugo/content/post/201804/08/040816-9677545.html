你當初是如何學會「操作系統」的？

我當年自學計算機啃了好幾個月的 C 然後發現工作中並用不上什麼。

很多學校裏面的課程都會讓你花一個學期的時間學 C，然後就覺得你啥都懂了，就開始教各種數據結構、算法、操作系統原理、編譯原理等。最後導致的問題是：你都不知道你學的這個是個啥。

說白了就是：你如果沒有辦法把正在學的東西和你已經掌握的知識結合起來，你就很難明白它在說什麼。

我當年自學踩過無數的坑，最後總結的學習方法是：

自頂向下學習法，首先學習你平時接觸最多的東西，然後帶着疑問去學習它背後的實現和邏輯。

  1. 先學 Linux 基本操作，譬如怎麼在電腦上裝一臺 Linux 出來、怎麼查看進程、怎麼操作文件、軟鏈硬鏈、格式化磁盤、配置網卡、curl 一個網頁等。你可能會問爲啥要用 Linux 呢，因爲你工作之後 90% 的公司都會強制要求你的代碼運行在 Linux 下，所以你必須具備 Linux 的操作能力。當年我看的是鳥哥，可以不用全部看完，當然現在應該很多這類書，可以自己找一下，知乎也有對應的回答 有沒有比《鳥哥的 Linux 私房菜》更好的書？ 目的都一樣，先把 Linux 基本操作掌握了。強烈建議如果你希望自己的操作系統方面的知識好一些，趕緊把你的 Windows 系統裝成 Ubuntu，或者換臺 Mac 也行。
  2. 當你 Linux 掌握的不錯的時候，用你的 Linux 環境去學一門簡單的編程語言，譬如 Python，當然你在學校裏面學了 C 和 Java 也可以。我當年看的是 《A Byte of Python》 中文英文都有，知乎也有很多相關的推薦 你是如何自學 Python 的？ 記住一定要在你的 Linux 或者 Mac 環境下來學習，此時你應該能夠做一些簡單的編程，同時見過一些基本概念，譬如單進程程序、多線程併發處理請求等等。你可能不是很理解裏面的一些概念，但是沒有關係，你學完後面的之後，你就能徹底明白了。
  3. 此時你可以開始學操作系統相關的概念了，《Linux 內核設計與實現》 是一本超級良心的介紹操作系統的書，用短短 300 頁把你之前所有的困惑全部梳理一遍。它不會講的特別細，但是它會把你帶入門。
  4. 之後你可以找一本介紹計算機組成原理的書了，同樣不會太厚，但是能帶你先入門。《編碼》 這本書就不錯，400 頁能告訴你爲什麼機器語言是二進制，CPU 是怎麼實現等等。
  5. 上面這兩本書雖然不是很厚，但是已經非常良心的刪減了很多不是必須知道的細節，可能需要反覆看多幾次才能理解，此時你會發現，原來我之前寫錯代碼的原因是這個。
  6. 算法、數據結構、數據庫等在這裏不介紹了。
  7. 此時，你應該已經入門了操作系統，同時具備基本的編程能力，你如果真把 3，4 點提到的書都看懂了，大學畢業後找到一個好工作應該不難（很多在職的工程師其實很多這些裏面的概念都不懂），之後就是看你自己發展方向，譬如 APUE、UNP、深入理解計算機系統都是好書，我覺得那個時候你應該能自己具備能力去尋找適合自己的書了。
