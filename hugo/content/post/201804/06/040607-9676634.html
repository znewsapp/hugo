我是從負責運輸公司的財務轉行成爲運輸的數據分析的。轉行大概準備了半年多，不知道能不能算迅速。

首先是數據 ** 處理 ** 能力，例如京東系統中發車數據、裝貨數據、客戶下單數據，可以用海量形容。巨大數據量不是 Excel 能處理的，因此需要有一定的編程技能（主要是 python 和 SQL，python 學習開發都簡單，SQL 用來直接數據庫提數。

此處硬廣推薦經典的 2 本動物書系列：

SQL 學習指南（第 2 版 修訂版） 和《利用 Python 進行數據分析》(Wes McKinney) ，大量實踐案例教會你如何利用 Python 庫高效解決各式各樣的數據分析問題。

其次是數據 ** 分析 ** 能力，有了數據還要理解數據本身代表了什麼，大數據內隱藏什麼知識，同時要結合“常識”判斷數據是否合理。因此需要理解業務。以運輸爲例，上海松江倉庫發車到全上海的時間，其實和崑山發車差不多，這個時候到底要不要在崑山建一個倉庫，然後把上海的倉庫租出去呢？發車是爲了保證時效一天發 3~4 班車，還是考慮成本一天發 1 班車呢？

物流業務硬廣：

《倉儲物流精細化管理全案（超值珍藏版）》 企業落實倉儲物流精細化管理的模板庫、範例庫和指導手冊。

《物流與供應鏈管理工具包》([英]格溫·理查茲) 超過 90 種工具，涵蓋運輸、倉庫和庫存管理，手邊必備工具書，助你隨時解決問題。

另外其他答案裏面提到數據可視化。數據可視化是一個很酷炫的藝術，但是數據可視化本身，要自己對數據有足夠理解，才能選擇出正確的圖表，才能把【正確的】信息以【正確的】方式傳遞給【正確的】人。

另外，同意這句觀點：希望成長爲「數據分析師」，就需要着手訓練自己的能力和洞察力。既然是「數據分析師」，那就分別從「數據」和「分析」兩方面入手。】、

順帶對比一下以前工作和現在工作重合和相似的地方讓大家感受一下，拋磚引玉。如有疑問歡迎評論或者私信溝通~

0、行業基礎知識積累

至少要了解行業相關專有名詞，比如物流行業國家標準《物流術語》 GB/T 18354-2006

多和業務條線的同事聊天，積累基礎知識，多認識人，避免轉行以後一臉懵逼一臉抓瞎啥都不會

1、對業務理解：

財務階段：主要從合同、資金流、業務實質、合規、如何入賬如何報稅方面瞭解業務，對業務實質理解基本靠合同。是否做某個業務，應該怎麼做，應該如何做，說實話和財務工作內容還是有差距的。

提升方向：多問幾個爲什麼。從整個業務線瞭解業務。比如運輸業務瞭解爲什麼要開設某條線路，爲什麼用大車，爲什麼某些線路裝載率偏低（例如返程車裝載率低，這是物流行業老大難問題），瞭解問題產生的原因，以及解決辦法 / 爲什麼沒有辦法解決。

分析階段：手上有 ABCDE.....N 種方法可以選，每種方法又有 EFGH....等各種優勢缺點，面對甲乙丙丁....情況會出現戊己庚辛...等問題，要權衡一二三四種方案，平衡各個部門利益，考慮找出（相對）最好的方案，然後走合同等流程，之後纔是後臺部門的事情。

2、IT 技能：

財務階段：Excel 至上，VLOOKUP 等各種函數玩的 666，甚至會一點 VBA。

分析階段：python 和 SQL 都要會。Excel 什麼的都是基礎技能。

3、對數據的敏感性：

財務階段：橫向對比，華東和其他大區單均差距多少，各個部分費用差距多少。環比上月哪一項費用漲了，具體大概是哪個科目費用高了，挖一下日記賬知道一下什麼原因，總結做一版 PPT，然後問業務側原因。

分析階段：財務問之前就知道 XX 部門上個月招了 XX 個臨時工 XX 個小時工 XX 個學生工 XX 個正式工，買了 XX 輛貨車，開了 XX 條線路，關了 XX 條線路，AA 倉庫的東西搬到 BB 倉了，導致人工、折舊、運費費用上升 XXXX 元，短期成本上升 XX 元，長期看節約成本 XX 元。

4、人際關係和溝通能力：

財務階段：（郵件）AA 部門的 BB 數據煩請在今天下班前給我，CC 部門煩請按附件模版上報數據，預測 / 預算系統在 DD 日開啓請各個部門及時填寫。平均每【周】聯繫 5~10 人。

分析階段：“XXX 物流公司老丟件能不能把這個物流公司開了”，“小堯你要是把 XX 物流公司開了，這個線路就沒別的公司跑了你還開物流公司嗎？”“不開了不開了，約談一下吧”；“XXX 大哥，爲啥要招 Y 個臨時工，不考慮一下換成 Z 個小時工嗎？”；“XXX 大哥，咱成本還有能下降的空間嗎？”平均【天】聯繫 5~10 人。
